📋 ค่าที่ต้อง Copy และ Paste ใน Railway Dashboard
═══════════════════════════════════════════════════════════════

ขั้นตอนที่ 1: Copy DATABASE_URL
───────────────────────────────────────────────────────────────
1. ไปที่ Railway Dashboard → PostgreSQL service → Variables tab
2. Copy ค่า DATABASE_URL
3. วางที่นี่ (เพื่อตรวจสอบ):
   [วาง DATABASE_URL ที่นี่]

───────────────────────────────────────────────────────────────

ขั้นตอนที่ 2: Copy Backend URL
───────────────────────────────────────────────────────────────
1. ไปที่ Railway Dashboard → demona-backend service
2. Copy URL ที่แสดงด้านบน (เช่น: https://xxx.up.railway.app)
3. วางที่นี่ (เพื่อตรวจสอบ):
   [วาง Backend URL ที่นี่]

───────────────────────────────────────────────────────────────

ขั้นตอนที่ 3: ตั้งค่าใน Backend Service
───────────────────────────────────────────────────────────────

📍 DATABASE_URL:
   [วาง DATABASE_URL ที่นี่]

📍 ALLOWED_ORIGINS:
   [Backend URL],http://localhost:3000
   
   ตัวอย่าง:
   https://demona-backend-production-xxx.up.railway.app,http://localhost:3000

───────────────────────────────────────────────────────────────

✅ ตรวจสอบ:
   - DATABASE_URL ต้องเริ่มด้วย postgresql://
   - DATABASE_URL ต้องมี .railway.app (ไม่ใช่ db:5432)
   - ALLOWED_ORIGINS ต้องมี Backend URL จริง
